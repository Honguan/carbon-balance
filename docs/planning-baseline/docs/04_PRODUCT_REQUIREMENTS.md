# 產品需求文件（PRD）

## 1. 產品願景

讓企業以可理解、可協作、可查驗的方式完成產品碳足跡盤查，降低資料蒐集與係數選用負擔，同時保留足夠證據，使每一次計算都能重現、審核與追溯。

## 2. 目標使用者

| Persona | 需求 |
|---|---|
| 企業管理者 | 建立組織、產品、權限，掌握進度與結果 |
| 盤查專案負責人 | 設定 PCR、邊界、單位、資料期間與分工 |
| 內部資料提供者 | 填寫能資源、產量、運輸、廢棄等活動數據 |
| 供應商／合作廠商 | 僅填寫被指派資料並上傳證據 |
| 領域審核者 | 審查數據、係數、分配與例外理由 |
| 查驗準備人員 | 匯出清冊、報告、證據索引與變更紀錄 |
| 系統管理員 | 管理係數、PCR、組織、權限、保留與系統健康 |

## 3. 核心使用流程

```text
建立組織與成員
→ 建立產品及版本
→ 建立盤查專案
→ 選擇／確認 PCR 與系統邊界
→ 定義功能／標示單位、期間、產量與分配
→ 建立生命週期流程與資料需求
→ 指派資料提供者
→ 填寫活動數據與證據
→ 搜尋／選用排放係數
→ 執行驗證與資料品質檢查
→ 建立不可變計算版本
→ 內部審核與補正
→ 匯出清冊、報告與查驗包
→ 登錄查驗／核定結果
```

## 4. 功能範圍

### P0：第一個可用正式版本

1. 組織、使用者、角色與 MFA。
2. 產品、產品版本、廠場與盤查專案。
3. PCR 記錄、適用性、功能／標示單位、邊界與期間。
4. 五階段資料輸入：
   - 原料投入與原料運輸。
   - 製造能資源、產量、分配、廢棄物與委外運輸。
   - 配送銷售運輸。
   - 使用情境。
   - 廢棄處理情境。
5. 單位目錄與換算。
6. 係數搜尋、版本、來源與選用。
7. 計算引擎、逐項明細、階段總和與產品總和。
8. 資料驗證、缺漏清單與警告。
9. calculation run snapshot 與版本比較。
10. Excel/CSV 匯入、匯出與錯誤報告。
11. 盤查清冊、盤查報告摘要與證據索引。
12. Audit log、備份、還原與基本觀測。

### P1：查驗與協作強化

1. 供應商邀請與限縮權限。
2. 審核工作流、留言、補正與核准。
3. 資料品質評分與不確定性欄位。
4. PCR 規則模板與欄位動態要求。
5. 查驗包匯出、查驗發現與回覆。
6. 核定編號、期限與狀態管理。
7. 係數更新與差異通知。
8. 多語介面與報告。

### P2：分析與整合

1. 版本差異與熱點分析。
2. 減碳情境比較，不覆寫基準盤查。
3. 外部 ERP／MES／能源／採購資料匯入 API。
4. 主管機關或係數平台 API／批次同步（需確認授權與契約）。
5. 大量產品範本與企業主資料治理。

## 5. 非目標

第一版不包含：

- 自動宣告「通過查驗」或「取得碳標籤」。
- 取代查驗機構或自動簽發查驗聲明。
- 任意執行使用者輸入的公式程式碼。
- 完整組織型溫室氣體盤查平台。
- CBAM 申報或碳費計算；可在後續獨立模組規劃。
- 微服務、事件串流與跨區多活等未有需求證據的複雜架構。

## 6. 關鍵業務規則

1. 盤查專案綁定產品版本，不直接綁定可變產品主檔。
2. PCR 與係數選用都必須保存當時版本。
3. 活動數據需保存資料期間、原始值、單位、轉換值、來源與證據。
4. 共同能資源需保存分配基礎、公式、分母、分子與理由。
5. 不適用、排除、估算與替代資料都必須有理由及審核狀態。
6. calculation run 建立後不可變；任何修正建立新 run。
7. 報告顯示的四捨五入不得改變內部計算精度。
8. 撤回或更新的係數不影響歷史 run，但不得用於新 run，除非有核准例外。
9. 不同組織資料必須在資料庫查詢與授權層隔離。
10. 系統狀態與查驗／核定法律狀態必須分開。

## 7. 非功能需求

| 類別 | 需求 |
|---|---|
| 可用性 | 表單自動儲存草稿；錯誤指出欄位與修正方式 |
| 正確性 | 金額與碳排類數值不使用 binary floating point |
| 可重現 | 任一 calculation run 可由快照重新計算並得到相同結果 |
| 安全 | MFA、最小權限、CSRF、XSS、SQL Injection 防護、secret 管理 |
| 稽核 | 新增、修改、審核、計算、匯出、權限變更可追蹤 |
| 效能 | 一般表單 P95 < 1 秒；中型盤查計算 P95 < 10 秒（需以實際資料校準） |
| 可維護 | 模組邊界、ADR、migration、測試與支援版本政策 |
| 可攜 | Docker 化；開發與正式使用相同主要依賴版本 |
| 無障礙 | 以 WCAG 2.2 AA 為設計目標 |
| 國際化 | 文字、單位、日期與數字格式可本地化 |

## 8. 成功指標

- 完整盤查的必填資料缺漏可被系統自動列出。
- Golden Case 計算結果在允許精度內一致。
- 歷史 run 在係數更新後仍完全可重現。
- 所有報告數字可追到活動數據、係數、公式與來源。
- 沒有高風險未修正弱點即可發布。
- 新開發者可依 README 在 30 分鐘內啟動開發環境並通過測試。
